## Definition
**Self-RAG** is a RAG framework where the **same LLM both generates the answer and decides when to retrieve, evaluate, and integrate external knowledge** using special *reflection tokens*.  
These tokens control retrieval, judge evidence, and determine whether the generation should continue or revise.

---
## Core Idea
Traditional RAG:
$$
y = f_\theta(x, D_{retrieved})
$$
where:
- $x$ = user query  
- $D_{retrieved}$ = retrieved documents  
- $f_\theta$ = LLM

But retrieval happens **once before generation**. Self-RAG instead allows the model to **dynamically retrieve during generation**.

---
## Special Reflection Tokens

During training, the model learns to generate **control tokens** such as:

| Token | Meaning |
|------|------|
| `<RET>` | retrieve documents |
| `<ISREL>` | check document relevance |
| `<ISSUP>` | check if document supports the answer |
| `<FINAL>` | finalize answer |

These tokens are **generated by the same LLM** as part of the decoding process.

---
## Generation Process

Let the model be:
$$
p_\theta(t_i \mid x, t_{<i})
$$
where:
- $t_i$ = next token
- $x$ = prompt
- $t_{<i}$ = previous tokens

Tokens can be either:
- **text tokens**
- **reflection tokens**

---
## Retrieval Trigger

When the model outputs

$$
t_i = \langle RET \rangle
$$

the system:

1. Encodes the current context
2. Queries a retriever
3. Returns documents $D$

$$
D = Retriever(x, t_{<i})
$$

These documents are appended to the context.

---
## Evidence Evaluation

The model then generates relevance decisions.
Example:
$$
\langle ISREL \rangle(d_j) \in \{0,1\}
$$
If relevant:
$$
d_j \rightarrow context
$$
If not:
$$
d_j \rightarrow discard
$$
---
## Support Verification

The model checks if evidence supports the answer.
$$
\langle ISSUP \rangle(a, d_j) \in \{supported, partial, none\}
$$
Only supported evidence is used for final generation.

---
## Final Answer
Once sufficient support is found the model outputs:
$$
t_i = \langle FINAL \rangle
$$
and generates the final text answer.

---
## What Model Generates What Token?
**Single model:**
- Generates **normal language tokens**
- Generates **reflection control tokens**

The **retriever is external** and only activated when the model emits `<RET>`. Thus:

| Component | Role |
|------|------|
| LLM | generates answer + control tokens |
| Retriever | returns documents |
| Reranker / evaluator | often integrated into LLM |

---
## Step-by-Step Example

Query:

$$
x = \text{"When did Ada Lovelace publish the first algorithm?"}
$$

### Step 1 — Start generation
Model outputs:
```
Ada Lovelace published the first algorithm in  
```

### Step 2 — Retrieval
Retriever returns documents:
$$
D = \{d_1, d_2, d_3\}
$$
Example snippet:
```

d1: Ada Lovelace wrote the first algorithm in 1843

```

### Step 3 — Relevance Check
Model generates:

```
d1 → relevant  
d2 → irrelevant
```

### Step 4 — Support Verification
```
d1 supports claim
```

### Step 5 — Continue Generation
Model continues:
```
1843 while translating Luigi Menabrea's paper.
```

### Step 6 — Finalize

```
Final answer:

> Ada Lovelace published the first algorithm in **1843** while translating Luigi Menabrea's paper.
```

---
# Key Difference From Standard RAG
Standard RAG:
$$  
Retrieve(x) \rightarrow Generate  
$$
Self-RAG:
$$  
Generate \rightarrow Retrieve \rightarrow Evaluate \rightarrow Generate  
$$
interleaved during decoding.

---
# Summary
Self-RAG allows an LLM to:
1. **Decide when retrieval is needed**
2. **Evaluate document relevance**
3. **Verify factual support**
4. **Generate the final answer**

All through **tokens generated by the model itself**, making retrieval **adaptive and self-controlled**.
